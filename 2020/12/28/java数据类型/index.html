java数据类型

2020-12-28- 2021-01-02
java基础-日学记我不是咸鱼

是什么

顾名思义数据类型就是数据的种类，类似将具有不同属性的动物分类为猫或者狗一样，数据类型是一种人为的分类方式，将具有不同性质的数据分成不同的类。

为什么

引入数据类型是为了更好的将数据分类。我认为主要有两个方面的原因：

1.计算机无法自动判断一个数据应该成为的数据类型，比如12345这串数字，程序员究竟希望他呈现为int还是char亦或者double。

2.将数据分类并为每一种数据制定相应的规范，确认存储空间的大小、位置，可以极大的提升内存使用率、程序运行速率（在你选择了更适合的数据类型来存储你的数据时）

基本数据类型

everything is a object（一切都是对象）是java语言的标签，不过java中也并没有将所有数据类型定义为对象，而是提供了8种基本的数据类型以及它们对应的包装类型（对象）。

这是基于程序性能的考量。基本类型定义的变量是存放在栈中，比如int i = 1；。除了基本类型以外java中所有对象方法的参数都是在传递引用的地址值而非本身的值，这里面的存储逻辑是这样的：实例化一个对象比如new Integer(1)，这会在堆内存中开辟一个存储空间用于存储数据（1），但这个时候你无法使用它，因为你不知道它在哪（没有指向它的引用），所以你需要Integer x = new Integer(1);这个时候x就成为了这个对象的引用存放在栈中。栈的读写速度远高于堆了。再有就是基本类型定义的变量创建和销毁很快，而类定义的变量还需要JVM去销毁。

下面是八种基本数据类型

  • byte/8

  • char/16

  • short/16

  • int/32

  • float/32

  • long/64

  • double/64

  • boolean/~

    boolean 只有两个值：true、false，可以使用 1 bit 来存储，但是具体大小没有明确规定。JVM 会在编译时期将 boolean 类型的数据转换为 int，使用 1 来表示 true，0 表示 false。JVM 支持 boolean 数组，但是是通过读写 byte 数组来实现的。

包装类型

基本类型都有对应的包装类型（对象形式的基本类型），基本类型与其对应的包装类型之间的赋值使用自动装箱与拆箱完成。

缓存池

new Integer(123) 与 Integer.valueOf(123) 的区别在于：

  • new Integer(123) 每次都会新建一个对象；
  • Integer.valueOf(123) 会使用缓存池中的对象，多次调用会取得同一个对象的引用。

valueOf() 方法的实现比较简单，就是先判断值是否在缓存池中，如果在的话就直接返回缓存池的内容。

在 Java 8 中，Integer 缓存池的大小默认为 -128~127。

编译器会在自动装箱过程调用 valueOf() 方法，因此多个值相同且值在缓存池范围内的 Integer 实例使用自动装箱来创建，那么就会引用相同的对象。

      基本类型对应的缓冲池如下：

  • boolean values true and false
  • all byte values
  • short values between -128 and 127
  • int values between -128 and 127
  • char in the range \u0000 to \u007F

在使用这些基本类型对应的包装类型时，如果该数值范围在缓冲池范围内，就可以直接使用缓冲池中的对象。

在 jdk 1.8 所有的数值类缓冲池中，Integer 的缓冲池 IntegerCache 很特殊，这个缓冲池的下界是 - 128，上界默认是 127，但是这个上界是可调的，在启动 jvm 的时候，通过 -XX:AutoBoxCacheMax= 来指定这个缓冲池的大小，该选项在 JVM 初始化的时候会设定一个名为 java.lang.IntegerCache.high 系统属性，然后 IntegerCache 初始化的时候就会读取该系统属性来决定上界。
