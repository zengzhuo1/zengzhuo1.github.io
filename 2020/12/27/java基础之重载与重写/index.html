java基础之重载与重写

2020-12-27- 2021-01-02
java基础-日学记我不是咸鱼

是什么

重写

       子类对父类方法的重新实现，两个方法返回值、形参必须一致，主要为实现内容可以不同。

重载

       同一类中，同名方法，以参数、返回类型的不同进行区分（参数类型列表必须不同），针对不同参数采用不同的处理方式。

为什么

       使用这两者可以体现java多态性，使得方法更加整合复用。
       这样说可能比较抽象，可以这样理解它们的应用场景，java作为面向对象的编程语言一切都可以视作对象，对于同一类对象（例如用户），都有同一种动作（方法，例如发消息），但是依据不同的内容（参数）这一动作（方法）有不同的实现方式（例如发语音，发文字），这个时候就可以重载。对于该对象的子类（例如会员用户），它们的该动作（发消息），需要以与众不同的方式实现（例如发带特效的消息），这个时候就需要重写父类（用户）的方法（发消息）已实现不同的效果。
       总的来说使用重写与重载主要有以下两个好处
       １.不用写很多方法，只写一个方法就可以了。
       ２.对象调用方法的时候只要调用一个方法就可以了，虽然调用的是同一个方法，但是却因为不同的对象／不同参数的调用而产生不同的效果

怎么做

重写equals方法

       说了这么多那么具体重写应该如何操作呢，下面为将通过重写一个对象（有id，name，age三个属性）的equals方法来记录。equals方法是obj类的方法，比较两个对象的地址是否相等，在这里通过重写该对象的equals方法使得通过该对象的name，age属性判断是否相等。这里展现了核心方法（一般来说要一起重写hashcode）。

@Override
public boolean equals(Object obj) {
    if(this == obj){
        return true;//地址的对象自然相等
    }
    if(obj == null){
        return false;//非空性：对于任意非空引用x，x.equals(null)应该返回false。
    }
    if(obj instanceof user){
        user other = (user) obj;
        //当该对象name和age相等时，则这两个对象相等
        if(equalsStr(this.name, other.name)
                && equalsStr(this.age, other.age)){
            return true;
        }
    }
    return false;
}
